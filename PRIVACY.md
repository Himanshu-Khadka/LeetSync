# Privacy Policy

Effective date: June 28, 2026

LeetSync does not collect analytics, sell data, serve advertisements, or operate an external server.

## Data stored in your browser

LeetSync uses the browser's extension storage to keep:

- your GitHub personal access token;
- the GitHub owner, repository, branch, and destination folder you choose;
- your README preference;
- the latest sync status; and
- up to 50 recent LeetCode submission IDs for duplicate prevention.

This data remains in your browser profile unless browser synchronization or device-management software copies extension storage. Removing the extension normally removes its local data. You can remove the saved GitHub token at any time from the popup.

## Data sent over the network

LeetSync communicates directly with two services:

- **LeetCode:** while you are signed in, the extension asks `leetcode.com` for your recent submission result, accepted source code, and basic problem metadata.
- **GitHub:** the extension sends your GitHub token to `api.github.com` to check repository access and commit your accepted source code and generated metadata.

LeetSync does not send this information to its developers or to any other destination. LeetCode and GitHub process data under their own privacy policies.

## Permissions

The extension requests storage access and host access to `leetcode.com` and `api.github.com`. These permissions are used only for the behavior described above. LeetSync does not execute remotely hosted code.

## Changes

Material changes to this policy will be documented in the project changelog and released with an updated effective date.

Questions and security reports should be opened through the repository's issue tracker without including access tokens or private solution code.
